Benni's GNU/Linux C/C++ Toolchain for Windows64:

This toolchain brings the bright Linux64 C/C++ development world to Windows64.
U will never need a Microsoft IDE or Microsoft compiler to build fully functional and high performance Windows apps.
All is done using free available and platform independent tools:
CMake, MinGW compiler, git4windows and NASM assembler.
Any multimedia app and library ( e.g. Steinberg VST Audio Plugins )
is possible using only (mostly) free software

Advantages of my Toolchain:

  • DAU safe, easy to read, easy to understand and to contribute to.

  • Colored console messages (e.g. errors are red). Surprises Visual Studio users everytime, even graphics programmers.

  • A terminal with history ( remembers last used commands ) and auto-completion.

  • Uses C++17 standard as default ( has std::filesystem, smart pointers, make_unique<>, etc.. )

  • Only 3 programs have to be installed on Windows64 (QtSDK, CMake, git4windows)

    1. QtSDK
      • The QtSDK is a real life saver and the most important program package in the last 20yrs
      • brings QtCreator, the IDE i daily use since 2016 (Visual Studio replacement).
      • brings MinGW64 GNU C/C++ compiler suite, atleast version 7.3.1 x64
        • contains gcc.exe (C compiler)
        • contains g++.exe (C++ compiler)
        • contains other compilers like fortran, a GNU assembler, etc...
        • contains gdb.exe (Debugger)
        • contains windres.exe (RC resource compiler, so u can develop win32 dialogs and embed window icons)
        • contains mingw32-make.exe (make replacement)
          • I dont use MSYS anymore.
          • I dont use CYGWIN.
          • I dont use MSYS2, but git4windows probably does itself.
      • brings precompiled dlls for graphical user interface development ( aka QtCore, QtWidgets, etc... ).

    2. CMake, atleast version 3.14
      • All libraries, applications and assembler inside the Benni toolchain are built with custom CMake recipes.

    3. Git4Windows ( my daily used ANSI colored terminal + git )
      • This program also brings /etc/bash.rc to customize your terminal promt.

  • You can develop any multimedia program (audio,video,3D and network) on Win64 using free GNU/Linux tools

  • Includes NASM.exe assembler by source

    • Used to build high performance audio and video codec libs (e.g. x264, x265 )
    • I want to be able to build FFMPEG (libAV) with my toolchain on Win64, but FFMPEG itself uses 100+ other (complicated and high perf) libs.

  • 99% of libraries are included by source code (e.g. jpeg8d, png, zlib, boost, CGAL, vst2sdk).

    • No incompatible libs ( due to different compile and link options )
    • Full control, easier debugging/profiling and custom UnitTesting.
    • Bug fixing without having to rely on the vendor to update precompiled libs
    • Increased security ( your custom changes lead to different libs, making fix address attacks unlikely )
    • Minimal project size, text is compressed much better than dlls.
    • High portability.
    • Less debugging due to less differences of runtimes.

  • 1% of libraries that are NOT included by source code.

    • FFMPEG is included by precompiled shared libs (still, to be changed).
    • wxWidgets is included by precompiled shared libs (still, to be changed).
    • libFMOD is included by precompiled shared libs, because its a proprietary, closed source lib.
    • QtWidgets is included by precompiled shared libs (static linking is forbidden without license, proprietary libs).

  • I dont use Microsoft Visual Studio to develop C/C++ programs for Windows.

    • Developer does not waste time with multiple runtimes (MT,MTd,MD,MDd)
    • This alone increases your productivity by 100%
    • Developer has a better terminal (with colors), not the worst-case cmd.exe
    • Developer uses same codebase and tools for Linux and Windows
    • The VS dialog to add libraries is the same nightmare since 1998.

  • I dont use configure scripts

    • Configure scripts are slow and single threaded, which would increase build times.
    • My toolchain is 100% of time at 100% CPU usage, meaning minimal build time.
    • Time is money.

  • I use CMake as project files

    • Platform independent tool for Win,Linux,MacOS
    • Adding your custom libs and apps is just a matter of copying and adapting a preset.
    • My minimal apps and various libs gives you a preset for any existing lib and project type.
    • QtCreator (my IDE) can open CMake projects
    • VisualCode can open CMake projects
    • CMake files are programs themselfs
    • CMake is much easier than makefiles
    • I only use a small subset of CMake commands to make it easy and reliable.
    • VS project-files are not programmable.
    • Qt project-files are not programmable.
    • Eclipse project-files are not programmable.
    • Codeblocks project-files are not programmable.

Disadvantages of my Toolchain:

  • I dont invest much of my finite lifetime in Apple closed systems (MacOS, iOS).
  • I dont invest much of my finite lifetime in Google closed systems (Android).
  • No MFC, ATL since its M$ only. But you have win32Api and windres RC resource compiler
  • A small learning curve

Benni's additional daily used tools

7zip

The best file compressor/decompressor for me.
I dont use winrar or the like
My programs ofcourse still use zip, bzip2, because much depend on them.
But you wont be able to test any of my programs without 7zip since they are all compressed that way.

Notepad++

The old notepad.exe is the next ugly program that never got better at anything.
Replacing it is a must for developer.
Notepad++ can syntax highlighting and can convert between encoding like UTF-8 with or w/o BOM
Could use some plugins to better understand shaders (glsl).

IrfanView

Must have for graphics programmers. Best and fastest image viewer, easy to say because the competition is just really bad. Shows hex colors of pixels on mouseclick, can resize and rotate images. Does handle transparency well, but not perfect. Has Atleast 100x faster startup than gimp or such monsters. Can view many formats, not all. Can export many formats, not all ( e.g. .xpm missing ) Can decompose gif animations into its frames. Views .ico files good enough. Has many filters and counts unique colors.

ConEmu

Another ANSI terminal emulator on Windows. It works but i simply use the terminal in Git4Windows.

ASIO4ALL free audio driver

A must have for audio developers and producers with generic soundcards. I install this driver on any of my Windows machines, to develop realtime audio with RtAudio mostly. I currently dont have any special ASIO hardware.
I used to have more professional Audio hardware when i was developing at AlcaTech GmbH in Dresden

OpenAL audio driver

Some apps and libraries depend on it, e.g. libSFML. Is yet another audio driver needed for some apps, its kinda easy to program with but latencies are not really low, because it uses DSound itself asfar as i understand with almost 120ms latencies. Enough for playing music in games, but not low enough for producing music.

VLC

Can play almost any video, audio and image file. Based on FFMPEG (libAVCodec) and QtSDK. My daily video/DVD viewer and music player. Didnt test Blue-Ray since a long time.

Handbrake

Ok video converter. Sadly does not extract audio only.

ClipGrab

Download high resolution videos from Youtube. My browser plugins stops at 720p. Anything higher is grabbed using this tool. Based on yt-download.

Virtual-TI / ti-emu

My desktop calculator app. You need to own a Texas Instrument Ti89 or higher to be able to use it. Since u need a ROM downloaded from the calculator. Sharing is forbidden.
