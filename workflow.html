My C++ workflow:

Wake up PC and myself:

  • Brew filter coffee (10min)
  • Start Windows 10 x64 PC ( i regularly get bluescreens at startup because Win10 is shit )
  • Login

Develop C/C++ on Windows64:

1. Start linux terminal on windows (_dev_with_gitbash.bat script) 1.1. Navigate to workspace (Open explorer and goto desired directory)
1.2. Double click the .bat Batch script that starts the developing environment
1.3. It opens 2 windows, the cmd.exe window in background and the good git-bash terminal in front
1.4. Minimize the cmd.exe window and never think about it again
1.5. Move and resize the git-bash terminal window to your favourite pos and size.
2. Build (start one of the build scripts next to .bat file) 2.1. Press STRG+R to open terminal history
2.2. Type 3 letters bui to find last build command in history
2.3. Press ENTER and building starts...

or if u never built before

e.g. ./build_win_release_shared.sh and press ENTER

e.g. ./build_win_debug_shared.sh and press ENTER
e.g. ./build_win_debug_static.sh and press ENTER

e.g. ./build_win_relwithdebinfo_static.sh and press ENTER
3. Execute program 3.1. Press STRG+R to open terminal history again

My C/C++ developing machine:

  • AMD Ryzen 5 PRO 4650G ( 6C12T 4.3GHz all core beast, still energy saving <120W, with modern graphics card )
  • As a graphics programmer its good to have a slow graphics card so u can feel your bad/unoptimized programming.
  • As a graphics programmer its good to have a modern graphics card so u can develop for advanced features,shaders and extensions.
  • Atleast 8GB RAM, normal would be 16GB, 32GB is better for very big projects like a custom Linux System
  • Normal would be 16GB DDR4 ( my case )
  • 32GB and above is only needed for extreme projects (e.g. Technisat navigation system did not compile with only 16GB RAM)
