MIDI Parser v1 (C++)

2023

This C++ project parses binary (.mid) MidiFiles. The file extension is not really important.
The MidiParser class (producer) registers MidiParserListener classes (consumer).
Every consumer derives from the listener class and overrides only the methods it really needs.
The first listeners were a ConsolePrinter and a HtmlPrinter in v1.
The second listener in v2 is the MidiFileListener that fills the structs defined in MidiFile.hpp. Its my first attempt to completly parse all stuff contained in a MidiFile.
The listener still misses special method for SysEx messages [mpSysEx()] and some other. The Parser definetly knows what SysEx messages are and emits them using mpToken().
I could have parsed into my MidiFile struct directly. But i wanted to have a clean interface to enable other programmers using my parser.
The HTML converter acts on the MidiFile.hpp data structs and creates images + html from it.
  • de-win32api ( small library that links all win32 libs and has some addiditional functions )
  • libDarkImage ( one of the most recent versions, since MIDI Parser is a recent project )
I like to parse MIDI files because they contain so much musical information in small space. , aka binary sheetmusic.
I just have to convert to some graphical representation to play them on my Piano.
Later i like to be able to play them with one of my AudioApps. For example using a custom General MIDI VST plugin or simply a synthesizer from my Benni DSP stuff in AudioSynth-qt.
So this workspace is again completly standalone and not interested in what libraries u have or have not installed on your system.
!!! Beware that the parser can end in an endless loop when it does not find trackheaders when it expects them.
!!! Please start the program using a console so u can see if the parser is working ok and be able to abort the app quickly if not.
MidiParser_v1 is the first Parser generation that converts to HTML text. Use it to debug where v2 failes.
!!! Beware that opening every 1MB of HTML can lead to 100MB RAM consumption!
!!! I had a 30MB html file that needed 2.3GB RAM and atleast one full minute of loading with Edge and Opera.
MidiParser_v2 is the program that creates images from notes and controllers, if any. It does not create GuitarTabs for classical sheetmusic (yet)
!!! The program is still in development, no guarantee whatsoever.
The images are created using my libDarkImage using ImagePainter class and builtin Font5x8.
