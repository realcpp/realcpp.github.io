WGL_ShaderArt (C++, Desktop OpenGL WGL, GLEW, GLM, Font5x8)

2023

Uses WGL hardware accelerated OpenGL Window on Windows64 with 10+ example shaders.
The shaders rely on gl_VertexID ( kinda modern ), so i dont need any VAO, VBO or IBO to draw a screen quad.
In Release Static mode it produces single exe files, which is what i desired.
Offers easy to use events using another class IEventReceiver, which is platform independent.
I translate Windows Keys to platform independent key. So i will later be able to add Window classes for other operating systems
The full German keyboard is working with Umlauts, which is already better than glfw and Qt.
I will do audio synth apps ( still to come ) with this work and have more PianoKeys avail.
Uses Desktop OpenGL (WGL), no EGL or GLES dlls needed.
WGL is really bad and quirky. I needed to create a second (dummy) window to make opengl work.
No MSAA multisampling antialiasing so far by WGL, i will implement it myself with FXAA shader.
WGL_EXT_swapinterval was not loading and working, even though it is offered as WGL extension.
Maybe this is a Windows 8+ only thing because of DWM, i use DwmFlush() as replacement.
  • WGL ( Desktop OpenGL 4.6 on Windows64 )
  • GLEW ( loads all OpenGL commands up to 4.6 at end of WGL window creation phase).
  • GLM ( OpenGL math types, used in IShader/GL_Shader to set uniforms ).
  • Font5x8 ( Builtin font to draw text ).
This is a fully standalone project that everybody can build without having to rely on find_package() or what you have or have not installed on your system.
The tutorials should interest any Windows 3D graphics programmer.
Projects does not need media files. Aka minimal
